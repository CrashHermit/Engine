# Worldgen Redesign Plan

Status: **planning — awaiting confirmation before implementation.**

Scope: `src/worldgen` only — a pure map generator (seed + config in, `WorldData` out).
`service/world.py`, persistence, the TUI create-world modal, and the starter dungeon are
explicitly out of scope this round (they are currently broken against the new worldgen
shape and stay that way until their own round).

---

## 1. Decisions (interview log)

| # | Decision |
|---|----------|
| 1 | The world is a **simulation substrate** (narrative backdrop now, simulation later). Fields must be causally consistent, not just pretty. |
| 2 | **Keep the mesh/grid duality.** Voronoi mesh = generation-time physics domain (ephemeral). Square grid = gameplay base that subgraphs hang off. A future `region_id` tier (geography-aware clustering of mesh cells) gets a socket, not an implementation. |
| 3 | **Data layout: typed dataclass-of-arrays** (`MeshFields` / `GridFields`, numpy arrays indexed by cell/tile id). Kills the `MeshCell` ↔ `GridPositionData` parallel-struct duplication. Mesh→grid bake is one generic nearest-cell gather over all fields. Biome weights and magic channels are dense matrices. |
| 4 | **Terrain = plate-sketch uplift + fluvial erosion**, replacing noise-as-terrain. Anchors provider and `AnchorConfig` are deleted. |
| 5 | **World topology: torus** (both axes wrap). Latitude is replaced by an **authored insolation field**: one hot ring ("sunband") and one cold ring ("frostbelt"), an explicit config concept. All periodic machinery (4D noise wrap, 3×3 Voronoi tiling, grid wrap) stays. |
| 6 | **Climate: authored wind + simulated moisture transport.** No atmospheric circulation sim — ruled out permanently (runtime weather is a separate future system that consumes these fields as baselines). |
| 7 | **Hydrology: rivers and lakes are first-class objects** with discharge (precipitation-weighted), flow direction, and speed. |
| 8 | **Biomes: `BIOME_GRID` (core model) is the single source of truth.** IDW soft weighting (ecotones) kept; centers derived from band midpoints; `biome_centers.py` deleted. |
| 9 | **Savagery: terrain-derived** (remoteness + climatic harshness + ruggedness + noise). |
| 10 | **Alignment is replaced by the magic field model**: valence (pure↔corrupt) × channel triad (corpus/mens/anima, mirroring `Channel` in `src/core/model/threat.py`) × strength, generated by a **leyline/nexus network**. |
| 11 | **Validation: TUI viewer ported early** (primary tuning instrument), pytest invariant suite, fixed-seed PNG smoke renders. No golden-image tests. |

Known-broken things this plan fixes in passing: `alignment.py` SyntaxError
(`sample(x=x, x=y)`), dead `LakeBasin`, duplicate `drainage` field, global
`opensimplex.seed()` state.

## 2. Output contract

```python
@dataclass
class WorldData:
    seed: int
    size: int
    config: WorldgenConfig          # resolved snapshot
    grid: GridFields                # all per-tile arrays, shape (size*size,)
    rivers: list[River]             # path objects (see §5)
    lakes: list[Lake]               # basin objects
    leylines: LeylineNetwork        # nexuses + lines (see §7)
    landmasses: list[Landmass]      # id, size, class
```

`GridFields` (all numpy, one declaration per field, auto-baked from mesh):

- terrain: `elevation` (signed, **[-1,1], 0.0 = sea level**, piecewise-normalized:
  ocean depths → [-1,0), land → (0,1]), `is_land`, `landmass_id`, `landmass_class`,
  `coast_distance`, `slope`
- climate: `temperature`, `precipitation`, `wind_u`, `wind_v`, `wind_magnitude`
- water: `is_river`, `river_id`, `discharge`, `flow_u`, `flow_v`, `flow_speed`,
  `is_lake`, `lake_id`
- ecology: `biome_weights` `(n, n_biomes)`, `savagery`
- magic: `magic_strength` [0,1], `magic_valence` [-1,1], `magic_channels` `(n, 3)`
  (corpus/mens/anima weights)
- socket: `region_id` (constant -1 this round)

Mesh-side intermediates (uplift, plate ids, moisture, raw flux, flow tree) are
ephemeral but remain inspectable by the viewer, which runs the pipeline in-process.

## 3. Terrain

**Plate-sketch uplift** (one-shot cartoon of tectonics, no plate motion over time):

1. Partition mesh cells into ~8–15 plates (seeded coarse Voronoi flood-fill over
   the existing adjacency graph).
2. Each plate: random drift vector + type (continental = high base uplift,
   oceanic = low). Continents emerge as unions of continental plates.
3. At plate boundaries, compare drift vectors: convergent → elongated high-uplift
   belt (mountain range, smeared a few cells wide with falloff + noise);
   divergent → rift depression; transform → mild noise.

Uplift authorship sits behind a minimal provider seam so a pure-noise uplift can
exist as a fallback/debug option.

**Erosion — implicit stream power (Braun & Willett / FastScape core):**

- Per iteration: route flow (steepest descent receivers), order cells from base
  level upward along the flow tree, solve `z' = (z + dt·U + dt·K·A^m / d · z_rcv') / (1 + dt·K·A^m / d)`
  per cell (n=1 closed form), unconditionally stable, O(n) per pass.
- **Depression routing via priority-flood** (Barnes): computes spill levels and
  routes flow across bowls **without mutating elevation**. Remaining bowls at the
  end become lakes with real spillways.
- **Hillslope diffusion** each iteration to round ridges.
- Converges in tens of iterations at ~12k cells; uniform rain (drainage area) is
  the correct erosion driver — real rainfall weighting happens later (§5).

**Finalize:** sea level by land-fraction percentile on the equilibrium terrain →
piecewise signed normalization → `is_land`, landmass labeling (BFS components +
size classes), coast distance, slope. The old separate `SeaLevelStage`/post-erosion
re-cut disappears into this single stage.

## 4. Climate

- **Insolation field** (config-owned): cosine band over torus y — hot ring at
  phase 0, cold ring opposite. Single ring pair by default (`bands=1`).
- **Temperature** = insolation − lapse-rate elevation cooling, with **maritime
  moderation**: damp toward insolation mean by a factor decaying with
  `coast_distance` (continental interiors get extremes, coasts stay mild).
- **Wind (authored)**: belt pattern derived from the insolation rings (rise at hot
  ring, sink at cold ring, mirrored circulation between) + FBm turbulence +
  **terrain deflection** (steer around high elevation gradients, channel through
  passes).
- **Precipitation — moisture transport** (~20–40 cheap passes over the mesh):
  ocean cells evaporate `f(temperature)`; moisture advects downwind cell-to-cell;
  each step a fraction precipitates (base rate, sharply amplified by upslope
  motion and cold); remainder travels on. Emergent: wet windward coasts, dry
  interiors, real rain shadows, lush sunband / arid frostbelt.

---

## 5. Water

- **Discharge pass** (after climate): one O(n) sweep down the final flow tree
  accumulating `precipitation` instead of `1.0`. Deserts get sparse rivers fed by
  distant mountains; wet coasts get dense networks.
- **River objects**: walk the flow tree from each above-threshold source; path of
  cells to mouth (ocean or lake), per-step discharge, `tributary_of` where it
  joins a larger river (larger discharge keeps the identity). Nameable,
  traversable.
- **Lake objects**: basin cells, water surface level (from priority-flood), outlet
  cell, outlet river. Rivers flow *through* lakes and continue from spillways.
- **Direction & speed**: per-cell flow direction from the tree; speed
  `∝ slope^a · discharge^b` (Manning-flavored, normalized [0,1]) — fast mountain
  streams, slow lowland rivers, near-still through lakes.
- **Grid rasterization**: stamp river *paths* (not loose segments) with
  discharge-scaled width; tiles get `river_id`, `discharge`, flow vector, speed.

---

## 6. Ecology

- **Biomes**: centers computed from `BIOME_GRID` band-cell midpoints (band
  breakpoints already in `core/model`). IDW soft weights over (temperature,
  precipitation) with `blend_sharpness` and `weight_cutoff` knobs; optional
  per-biome dominance multiplier table (all 1.0 by default). Dominant soft biome
  always agrees with the discrete band lookup.
- **Savagery** = weighted blend of remoteness (`coast_distance`), climatic
  harshness (distance of temp/precip from livable mid-ranges), ruggedness
  (slope), + FBm noise. Optional magic coupling knob, default 0.

---

## 7. Magic (leylines)

Field model per cell: `strength` [0,1] (how magical), `valence` [-1,1]
(pure ↔ corrupt; "neutral" emerges from low strength or balance), `channels`
3-weight composition over corpus/mens/anima (which way magic expresses —
flesh-warping wilds vs. mind-twisting reaches vs. soul-blight, and their pure
mirrors).

Generator:

1. Place ~10–30 **nexuses**: seeded, biased toward significant terrain (peaks,
   lake outlets, confluences, ring alignment).
2. Connect with **MST + a few loop edges** over nearby pairs; each line is a
   straight torus chord ("ancient circuit" look; lines may circle the world).
3. Assign each nexus valence and a channel blend — **valence clusters** (corrupt
   nexuses prefer corrupt neighbors → coherent blighted/sacred regions with
   contested borders), **channels mingle** freely.
4. Rasterize: `strength` = falloff from nearest line/nexus + low background noise
   floor; `valence`/`channels` = distance-weighted blend of nearby lines, drifting
   to noise far from the web.
5. The network itself (nexus positions/aspects, line endpoints) ships in
   `WorldData` as first-class data — future quest/dungeon hooks.

Replaces `SavageryStage` noise (§6) and `AlignmentStage` entirely. The
`core/model/environment/ecology/alignment.py` band model becomes a consumer-side
concern for the persistence round.

## 8. Architecture & hygiene

- **Pipeline**: explicit ordered stage list, `Stage.run(ctx)` kept. New order:
  `Mesh → Plates/Uplift → Erosion → TerrainFinalize → Insolation/Wind →
  Moisture/Temperature → Discharge/Rivers/Lakes → Savagery → Leylines →
  Biomes → GridBake`.
- **Module layout**: `worldgen/{fields.py, context.py, pipeline.py, config/,
  geometry/, noise/, terrain/, climate/, water/, magic/, ecology/, bake/}`.
- **RNG hygiene**: instance-based noise (`opensimplex.OpenSimplex(seed)` per
  sampler, no global `seed()`); per-stage sub-seeds derived as
  `hash(world_seed, stage_name)` so adding/reordering stages doesn't reshuffle
  unrelated randomness.
- **Config**: regrouped per new stages (`PlatesConfig`, `ErosionConfig`,
  `InsolationConfig`, `WindConfig`, `MoistureConfig`, `HydrologyConfig`,
  `BiomeConfig`, `SavageryConfig`, `LeylineConfig`, `MeshConfig`). Presets
  rewritten in the new vocabulary (plate count/type ratio, sea level, ring
  contrast) — `earthlike`, `archipelago`, `pangaea` survive as intents.
- **Deletions**: `anchors.py`, `AnchorConfig`, `biome_centers.py`, `LakeBasin`,
  `drainage`, fat `MeshCell`/`GridPositionData`/`GridTileData`, old
  `SeaLevelStage`/`ErosionStage`/`AlignmentStage`/`SavageryStage` (superseded).
- **Dependencies**: make `numpy` explicit in `pyproject.toml` (currently only
  transitive via `scipy`).

---

## 9. Validation

- **Viewer first**: port `scripts/view_worldgen.py` / `worldgen_render.py` to
  `GridFields`/mesh fields immediately after the skeleton exists; add layers for
  every intermediate (plates, uplift, raw eroded z, moisture, discharge, magic
  strength/valence/channels, leylines, lakes).
- **Invariant tests** (pytest, seed-parameterized): determinism (same seed →
  identical output, run twice); flow tree strictly downhill (post-spill levels);
  every river terminates at ocean or a lake whose outlet chain reaches ocean;
  land fraction within tolerance; contract ranges (elevation [-1,1] & sea at 0,
  weight rows sum to 1, strength/valence in range); torus continuity across both
  wrap seams.
- **Smoke renders**: fixed-seed PNG exports for eyeballing; not compared
  automatically.

## 10. Phasing

| Phase | Deliverable |
|---|---|
| 0 | Field-storage skeleton (`fields.py`, context, pipeline shell), mesh build port, RNG hygiene, viewer ported with elevation layer, test harness |
| 1 | Plates → uplift → implicit erosion → finalize; viewer layers (plates/uplift/elevation); terrain invariants |
| 2 | Insolation, wind, moisture transport, temperature; climate layers |
| 3 | Discharge, river/lake objects, direction/speed, rasterization; water layers + invariants |
| 4 | Savagery, leylines/magic, biomes; layers |
| 5 | Final `WorldData` assembly, presets, full test suite, docs pass |

Each phase ends runnable and visible in the viewer.
